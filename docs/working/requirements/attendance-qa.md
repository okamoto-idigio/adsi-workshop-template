# 勤怠管理アプリ — 要求仕様 Q&A

## ユーザーストーリー（案）

### 社員として
- 出勤時に打刻したい（出勤時刻を記録するため）
- 退勤時に打刻したい（退勤時刻を記録するため）
- 自分の勤怠履歴を確認したい（月ごとの出退勤を振り返るため）

### 管理者として
- 部下の勤怠一覧を確認したい（勤務状況を把握するため）
- 勤怠データを CSV で出力したい（給与計算システムに連携するため）

## [Question] / [Answer]

### 1. ユーザー・認証

[Question] ユーザー（社員）はどのように識別しますか？（例: ログイン認証あり / 社員番号の手動入力 / 認証なしでシンプルに）
[Answer] ユーザー名だけでログイン。PW認証なしでシンプルに

[Question] 管理者と一般社員のロール（権限）分けは必要ですか？
[Answer] 管理者と一般社員のロールを作成

### 2. 打刻

[Question] 打刻方法は Web ブラウザからのボタン操作のみですか？（他に IC カード、GPS 等の手段は？）
[Answer]はい

[Question] 1日に複数回の出退勤（中抜け）は想定しますか？それとも 1日1回の出勤・退勤のみ？
[Answer] 1日1回の出退勤で中抜けなし

[Question] 打刻忘れの修正（手動入力・申請）機能は必要ですか？
[Answer] 不要

### 3. 勤怠データ

[Question] 勤務時間の計算ルールは？（例: 休憩時間を自動控除する / しない、残業の閾値は何時間？）
[Answer] 12時～13時を休憩時間として、自動控除する

[Question] 表示する期間の単位は？（日次 / 週次 / 月次）
[Answer] 月次

### 4. スコープ・優先度

[Question] MVP（最初のリリース）に含めたい機能はどこまでですか？（例: 打刻と一覧表示だけ / CSV出力まで / 管理者機能まで）
[Answer] 打刻と一覧表示だけ

[Question] 有給休暇・休日の管理は必要ですか？
[Answer] 不要

### 5. 技術

[Question] バックエンドは Java (Spring Boot)、フロントエンドは TypeScript (Next.js) で進めてよいですか？
[Answer] はい。

[Question] DB は何を使いますか？（例: PostgreSQL / MySQL / H2（開発用のみ））
[Answer] おすすめとメリット、デメリットを教えて
→ H2: セットアップ不要でワークショップ向き / PostgreSQL: 本番品質。どちらにしますか？
[Answer] H2で

### 6. 追加確認

[Question] 「ユーザー名だけでログイン」について: ユーザーは事前に登録済み（マスタデータ）ですか？それとも初回アクセス時に自分で登録しますか？
[Answer] マスターデータで。5つほど用意しておいて。

[Question] 管理者は「全社員」の勤怠を見られますか？それとも「自分の部署のみ」？（MVP では部署の概念は入れず全社員表示にすることも可能）
[Answer]全社員

[Question] 勤務時間の計算: 12:00〜13:00 をまたがない勤務（例: 9:00〜12:00）の場合、休憩控除は 0 分で正しいですか？
[Answer]はい

[Question] 打刻のバリデーション: 同日に出勤を2回押した場合はエラーにしますか？（退勤も同様）
[Answer] はい。
